Low resolution -> Basse resolution
It makes resolution half both in horizontal and vertical direction. -> Diminue la résolution de moitiée dans le sens horizontal et vertical.
PNG (slower than JPEG) -> PNG (plus lent que JPEG)
JPEG is a lossy compression. It is fast and has a very good compression ratio, but the image gets noisy. PNG is much slower, the image size is larger, but it uses lossless compression. -> JPEG est une compression avec pertes. C'est rapide et possède un très bon rapport de compression, mais l'image devient bruitée. PNG est beaucoup plus lent, l'image est plus volumineuse, mais il n'y a pas de perte lors de la compression.
Use only first framebuffer -> Utiliser le premier framebuffer seulement
Usually Android uses double buffering: while the phone uses one of the two framebuffers to display, it prepares the new image on the other one. Then swaps them. That's an old trick to avoid flickering. On some older phones (for example on HTC G1 aka Dream) may display some parts from the first, and some parts from the second framebuffer, but in this case usually the second framebuffer doesn't contains a whole image. This option forces Webkey to use only the first framebuffer. -> Habituellement, Android utilise une double mise en mémoire tampon: alors que le téléphone utilise un seul des deux framebuffer pour l'affichage, l'image suivante est préparée sur l' autre framebuffer. Ils sont ensuite interchangés. C'est un vieux truc pour éviter le scintillement. Sur quelques téléphones plus vieux, dont le HTC G1 ou Dream, l'affichage peut se faire avec certaines parties du premier framebuffer et d'autres parties du deuxième, mais en général, le deuxième ne contient pas une image complète. Cette option force Webkey à utiliser le premier
Record images -> Enregistrement des images
The phone starts to save images to a temporary folder /sdcard/webkey_TEMP/ as fast as it can. It uses the selected options (Low resolution, Horizontal, Flip, Png/Jpeg, Use only first framebuffer). After pressing Stop recording, the phone makes a zip archive from the images and sends it to the browser. -> Le téléphone commence à enregistrer les images dans un dossier temporaire /sdcard/webkey_TEMP/ aussi rapidement que possible. Il utilise les options sélectionnées (Basse résolution, Horizontal, Miroir, Png/Jpeg, Utiliser le premier framebuffer seulement). Après avoir cliqué sur Arrêter l'enregistrement, le téléphone crée une archive zip à partir des images et l'envoie au fureteur.

The large text area is for inject keys to the phone. You can do it immediately if "Send while typing" is checked, or you can send all the written text by pressing the button Send text. The injection of every non-English character may reload your phone's screen at the first time. The button DEL injects Back. -> La grande boîte de texte est pour envoyer du texte au téléphone. Vous pouvez le faire en temps réel si "Envoyer en tapant" est cochée, ou vous pouvez envoyer tout le texte en utilisant le bouton "Envoyer le texte". SMS mode -> Mode SMS
If you try to send special characters, like ő, ű in an SMS message, then these characters force Android to switch to unicode format, where the number of characters in a message drops from 160 to 70 in GSM networks. If SMS mode is checked, then lines with "works in SMS mode" have priority. -> Si vous essayez d'envoyer des caractères spéciaux tel que ő ou ű dans un message SMS, Android utilisera le format unicode, réduisant le nombre de caractères de 160 à 70 sur les réseaux GSM. Si le Mode SMS est coché, les lignes avec "fonctionne avec SMS" ont priorité.
