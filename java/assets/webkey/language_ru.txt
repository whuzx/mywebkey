Low resolution -> Низкое разрешение
It makes resolution half both in horizontal and vertical direction. -> Это уменьшит разрешение в два раза
PNG (slower than JPEG) -> PNG (медленнее, чем JPEG)
JPEG is a lossy compression. It is fast and has a very good compression ratio, but the image gets noisy. PNG is much slower, the image size is larger, but it uses lossless compression. -> JPEG это сжатие с потерями. Это быстро и имеет очень хорошую степень сжатия, но изображение становится "шумным". PNG намного медленнее, размер больше, но он использует сжатие без потерь.
Use only first framebuffer -> Используйте только первый кадровый буфер
Usually Android uses double buffering: while the phone uses one of the two framebuffers to display, it prepares the new image on the other one. Then swaps them. That's an old trick to avoid flickering. On some older phones (for example on HTC G1 aka Dream) may display some parts from the first, and some parts from the second framebuffer, but in this case usually the second framebuffer doesn't contains a whole image. This option forces Webkey to use only the first framebuffer. -> Обычно Andriod использует двойной буфер: в то время как телефон использует один из двух буфера кадров для отображения, он готовит новый образ на другой. После кеширует их. Этот трюк, чтобы избежать мерцания. На некоторых старых телефонов (например, на HTC Dream G1) может отображать некоторые части от первой, и в некоторых частях со второй фреймбуфер, но в этом случае обычно второй фреймбуфер не содержит целого изображения. Эта опция заставляет Webkey использовать только первый кадровый буфер.
Record images -> Запись изображения
The phone starts to save images to a temporary folder /sdcard/webkey_TEMP/ as fast as it can. It uses the selected options (Low resolution, Horizontal, Flip, Png/Jpeg, Use only first framebuffer). After pressing Stop recording, the phone makes a zip archive from the images and sends it to the browser. -> Телефон начнет сохранять картинки в папку / SDCard / webkey_TEMP / так часто, как сможет. Он использует настройки (с низким разрешением, горизонтальные, Flip, PNG / JPEG, используется только первый фрейм-буфер). После нажатия кнопки Остановить запись, телефон делает почтовый архив с изображениями и отправляет его в браузер.

SMS mode -> SMS режим
If you try to send special characters, like ő, ű in an SMS message, then these characters force Android to switch to unicode format, where the number of characters in a message drops from 160 to 70 in GSM networks. If SMS mode is checked, then lines with "works in SMS mode" have priority. -> Если вы попытаетесь послать специальные символы, например, O, U в SMS, то эти символы заставить Android перейти на формат Unicode, где количество символов в сообщении падает с 160 до 70 в сетях GSM. Если SMS-режим включен, то линии "работать в режиме SMS» имеют приоритет.
