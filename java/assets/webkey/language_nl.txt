Low resolution -> Lage resolutie
It makes resolution half both in horizontal and vertical direction. -> Het maakt de resolutie half in horizontale en verticale riching.
PNG (slower than JPEG) -> PNG (langzamer dan JPEG)
JPEG is a lossy compression. It is fast and has a very good compression ratio, but the image gets noisy. PNG is much slower, the image size is larger, but it uses lossless compression. -> JPEG is een lossy compressie. Het is sneller en heeft een goede compressie ratio, maar de afbeelding krijgt ruis. PNG is veel langzamer, de bestandsgrootte is groter, maar is het een verliesvrije compressie.
Use only first framebuffer -> Gebruik alleen het eerste framebuffer
Usually Android uses double buffering: while the phone uses one of the two framebuffers to display, it prepares the new image on the other one. Then swaps them. That's an old trick to avoid flickering. On some older phones (for example on HTC G1 aka Dream) may display some parts from the first, and some parts from the second framebuffer, but in this case usually the second framebuffer doesn't contains a whole image. This option forces Webkey to use only the first framebuffer. -> Normaal gebruikt Android dubbele buffering: wanneer de telefoon gebruik maakt van één of twee framebuffers voor weergave, bereidt het de nieuwe afbeelding over de andere. En verwisseld ze dan. Dat is een oude manier om flikkering tegen te gaan. Op sommige oudere telefoons (bijvoorbeeld op HTC G1 aka Dream) kunnen delen worden weergegeven van de vorige, en sommige delen van de tweede framebuffer, maar normaal bevat de tweede framebuffer niet de hele afbeelding. Deze optie forceerd Webkey om alleen de eerste framebuffer te gebruiken.
Record images -> Plaatjes opnemen
The phone starts to save images to a temporary folder /sdcard/webkey_TEMP/ as fast as it can. It uses the selected options (Low resolution, Horizontal, Flip, Png/Jpeg, Use only first framebuffer). After pressing Stop recording, the phone makes a zip archive from the images and sends it to the browser. -> De telefoon start met het opslaan van de plaatjes naar een tijdelijke map /sdcard/webkey_TEMP/ zo snel mogelijk. Het gebruikt de geselecteerde opties (Lage resolutie, Horizontaal, Flip, PNG/JPEG, Gebruik alleen het eerste framebuffer). Nadat het indrukken van Stop opnemen, maakt de telefoon een aip van de plaatjes en zend ze naar de browser.

SMS mode -> SMS modus
If you try to send special characters, like Å‘, Å± in an SMS message, then these characters force Android to switch to unicode format, where the number of characters in a message drops from 160 to 70 in GSM networks. If SMS mode is checked, then lines with "works in SMS mode" have priority. -> Wanneer u speciale tekens probeerd te verzenden, zoals Å‘, Å± in een sms, dan kunnen deze tekens Android forceren om te schakelen naar unicode formaat, waar het aantal tekens in een bericht daald van 160 naar 70 in GMS netwerken. Wanneer SMS modus is aagevinkt, dan zullen de regels "werkt in SMS modus" voorrang hebben.
