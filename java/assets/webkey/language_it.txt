PNG (slower than JPEG) -> PNG (più lento di JPEG)
JPEG is a lossy compression. It is fast and has a very good compression ratio, but the image gets noisy. PNG is much slower, the image size is larger, but it uses lossless compression. -> JPEG è un formato lossy. E' veloce e ha un'ottima compressione, ma l'immagine perde in qualità. PNG è molto più lento, l'immagine è più pesante ma usa una compressione lossless.
Use only first framebuffer -> Usa solo il primo framebuffer
Usually Android uses double buffering: while the phone uses one of the two framebuffers to display, it prepares the new image on the other one. Then swaps them. That's an old trick to avoid flickering. On some older phones (for example on HTC G1 aka Dream) may display some parts from the first, and some parts from the second framebuffer, but in this case usually the second framebuffer doesn't contains a whole image. This option forces Webkey to use only the first framebuffer. ->  Di solito Android usa il doublebuffering: mentre il telefono usa uno dei due framebuffer per il display, prepara la nuova immagine sull'altro. Poi li schambia. Questo è un vecchio trucco per evitare il flickering. Su alcuni vecchi telefoni (per esempio l'HTC G1 o Dream) potrebbe mostrare alcune parti del primo e altre del secondo framebuffer, ma in questo caso di solito il secondo framebuffer non contiene l'immagine completa. Quest'opzione forza Webkey a usare solo il primo framebuffer.
Record images -> Salva immagini
The phone starts to save images to a temporary folder /sdcard/webkey_TEMP/ as fast as it can. It uses the selected options (Low resolution, Horizontal, Flip, Png/Jpeg, Use only first framebuffer). After pressing Stop recording, the phone makes a zip archive from the images and sends it to the browser. -> Il telefono inizia a salvare le immagini in una directory temporanea (/sdcard/webkey_TEMP/) quanto più veloce possibile. Usa le opzioni selezionate (Bassa risoluzione, Ruota di 90°, Ruota di 180°, Png/Jpeg, Usa solo il primo framebuffer). Dopo aver premuto Interrompi registrazione, il telefono crea un archivio zip dalle immagini e lo invia al browser.. 
Typing -> Digitazione
The large text area is for inject keys to the phone. You can do it immediately if "Send while typing" is checked, or you can send all the written text by pressing the button Send text. The injection of every non-English character may reload your phone's screen at the first time. The button DEL injects Back. -> La grande area di testo è per mandare caratteri al telefono. Si può fare immediatamente se "Manda durante la digitazione" è attivo, oppure mandare tutto il testo scritto premendo il bottone "Invia Testo". L'inserimento di caratteri non-inglesi potrebbe causare un refresh dello schermo del telefono la prima volta. Il bottone DEL manda "Indietro".

SMS mode -> Modalità SMS
If you try to send special characters, like ő, ű in an SMS message, then these characters force Android to switch to unicode format, where the number of characters in a message drops from 160 to 70 in GSM networks. If SMS mode is checked, then lines with "works in SMS mode" have priority. -> Se provi a mandare caratteri speciali come ő, ű in un messaggio SMS, questi caratteri forzano Android in modalità unicode, dovei l numero di caratteri in un messaggio scende da 160 a 70 nelle reti GSM. Se la modalità SMS è selezionata, le linee con "funziona in modalità SMS" hanno la priorità.
